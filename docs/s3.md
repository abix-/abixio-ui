# S3

Single authority for how abixio-ui uses S3. Covers SDK configuration,
every operation we call, how each feature appears in the UI, and response
handling quality.

Uses `aws-sdk-s3` (official AWS SDK for Rust). For server-side S3
compliance, see the abixio server repo's `docs/s3-compliance.md`.

---

## SDK Configuration

### What we set in `src/s3/client.rs`

| Config | Value | Rating | Notes |
|---|---|---|---|
| `behavior_version` | `BehaviorVersion::latest()` | 10/10 | Required. |
| `region` | User-provided or `us-east-1` | 9/10 | CLI defaults to us-east-1. No auto-detection. |
| `credentials_provider` | Static for auth, failing + `allow_no_auth()` for anonymous | 9/10 | Anonymous sends unsigned requests. |
| `endpoint_url` | User-provided | 10/10 | Required for non-AWS servers. |
| `force_path_style` | `true` | 10/10 | Required for MinIO, AbixIO, self-hosted. |
| `timeout_config` | connect: 10s, operation: 60s | 9/10 | Prevents UI hangs. |
| `app_name` | `abixio-ui` | 9/10 | In User-Agent header. |

### SDK defaults we rely on

| Config | Default | Rating | Notes |
|---|---|---|---|
| `retry_config` | 3 attempts, exponential backoff | 5/10 | Slow on bad endpoints. Should be 1 for connection tests. |
| `stalled_stream_protection` | Enabled | 7/10 | Protects against stalled downloads. |
| `response_checksum_validation` | Standard | 8/10 | Validates when server provides checksums. |
| `http_client` | hyper + rustls | 8/10 | Matches our TLS stack. |

### Configuration rating: 9/10

## UI Feature Details

### Object Detail Panel

| Section | Source | Description |
|---|---|---|
| Overview | `head_object()` | Size, content type, last modified, ETag |
| Storage | -- | Bucket, key |
| HTTP Headers | `head_object()` | All headers including `x-amz-meta-*` |
| Tags | `get_object_tagging()` | List + add/remove. Max 10. |
| Versions | `list_object_versions()` | ID, date, size, latest badge, restore, delete |
| Preview | `get_object()` | First 4KB as text |
| Actions | -- | Download, Share, Copy, Move, Rename, Delete |
| AbixIO | `/_admin/object` | Shard inspection, manual heal |

### Bucket Detail Panel

| Section | Source | Description |
|---|---|---|
| Overview | -- | Name, prefix, counts |
| Versioning | `get_bucket_versioning()` | Status + toggle buttons |
| Bucket Tags | `get_bucket_tagging()` | List + add/remove |
| Policy | `get_bucket_policy()` + `put_bucket_policy()` + `delete_bucket_policy()` | Inline JSON view, edit, create, delete |
| Lifecycle | `get_bucket_lifecycle()` + `put_bucket_lifecycle()` + `delete_bucket_lifecycle()` | Inline XML view, edit, create, delete |
| Actions | -- | Refresh, Delete Bucket |

### Presigned Sharing

Share button in object Actions opens modal:
1. Select expiry (1h / 6h / 24h / 7d)
2. Generate URL via `presign_get_object()`
3. URL displayed in selectable text field

### Sync Planning Scaffold

The repo now includes a sync-oriented recursive listing helper for the upcoming
sync engine:

- `list_objects_recursive_for_sync()`
  returns normalized object records for the future diff planner

This is not a shipped sync feature yet. It exists to support the new sync
section scaffold and the upcoming real diff planner.
